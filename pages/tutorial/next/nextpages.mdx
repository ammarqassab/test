# ما هو مفهوم `Next.js Pages` ؟

هو عبارة عن نظام الأساسي لإنشاء الصفحات و تحكم في مسارات التطبيق حيث تكون الصفحة عبارة عم ملف بإمتداد `js, .jsx, .ts, .tsx` 
حيث يكتب أسم الملف دوماً بحروف صغيرة و داخله نكتب مكون و نعلم أن أسم المكون دوما يبدأ بحرف كبير 
, أسم الملف يدل على أسم مسار الصفحة و لا يشترط أن يكون أسم ملف الصفحة بنفس أسم المكون مثال ...

```
// موقع ملف الصفحة في المشروع
pages/index.js

// المكون
function Home() {
return <div>Home</div>;
}

export default Home

// مسار الصفحة
http://localhost:3000/
```
# طريقة ترتيب الصفحات في التطبيق :

طريقة ترتيب ملفات الصفحات في مجلد `Pages` تحدد شكل مسار الصفحة لنأخذ أمثلة على ذلك .

```
// موقع الملف
Pages/index.js
// مسار الصفحة الرئيسية
http://localhost:3000


// موقع الملف
Pages/about.js
// أو ننشئ مجلد بأسم الصفحة حول و نضع داخله ملف الأندكس فالطريقتين متكافئتين
Pages/about/index.js
// المسار
http://localhost:3000/about


// موقع الملف
Pages/tutorial.js
// أو ننشئ مجلد بأسم الصفحة و نضع داخله ملف الأندكس فالطريقتين متكافئتين
Pages/tutorial/index.js
// المسار
http://localhost:3000/tutorial


// موقع الملف داخل مجلد دورات تعليمية
Pages/tutorial/next.js
// أو ننشئ مجلد بأسم الصفحة و نضع داخله ملف الأندكس فالطريقتين متكافئتين
Pages/tutorial/next/index.js
// المسار
http://localhost:3000/tutorial/next


// موقع الملف داخل مجلد دورات تعليمية
Pages/tutorial/html.js
// أو ننشئ مجلد بأسم الصفحة و نضع داخله ملف الأندكس فالطريقتين متكافئتين
Pages/tutorial/html/index.js
// المسار
http://localhost:3000/tutorial/html
```
هذه الطريقة تؤدي إلى إنشاء صفحات ثابتة تتحول إلى ملفات `html` عند تحويل المشروع إلى الإنتاج عند الرفع على استضافة

# إنشاء صفحات بمسارات ديناميكية :

عندما نريد إنشاء صفحة واحدة لمسار متغيير نتبع هذه الطريقة من أمثلة على هذه الصفحات صفحات المنتجات التي تأتي من الباك إند

```
// موقع الملف
pages/posts/[id].js
// المسار
http://localhost:3000/posts/1
أو
http://localhost:3000/posts/123
أو
http://localhost:3000/posts/1x15g
أو
.
.
```

في مجلد `pages` هناك مجلد يسمى `api`  له علاقة بصنع `api` داخلي , سيتم شرحه في قسم لاحق

# مفهوم التقديم المسبق `Pre-rendering` :

يجلب `Next.js` الصفحات قبل طلب المستخدم لها مما يجعل التطبيق سريع جداً حيث يمكن تخزين الصفحات التي تم إنشاؤها بشكل ثابت مؤقتًا بواسطة `CDN` بدون تكوين إضافي لتعزيز الأداء , هذا يعني إن `Next.js` ينشأ صفحات `html` في وقت الإنشاء ويجلبها قبل طلب المستخدم لها مما يحسن محركات البحث `SEO`

# هناك نوعان لمفهوم التقديم المسبق `Pre-rendering` :

* `Static Generation (Recommended) = SSG` : يتم إنشاء صفحات `html` ثابتة في وقت الإنشاء وستتم إعادة استخدامه عند كل طلب , أي يتم إنشائها الصفحات من قبل الطلب
* `Server-side Rendering = SSR` : يتم إنشاء الصفحات `html` عند طلب المستخدم لها , أي يتم إنشاء الصفحة في كل مرة يطلبها المستخدم

نوصي باستخدام طريقة `SSG` لأسباب تتعلق بالأداء و محركات البحث `SEO` حيث يمكن تخزين الصفحات التي تم إنشاؤها بشكل ثابت مؤقتًا بواسطة `CDN` بدون تكوين إضافي لتعزيز الأداء
